# User Profile — Developer

## Workflow Preferences
- Conventional commits: `feat(scope):`, `fix:`, `docs:`, `chore:`
- One concern per commit, focused changes
- Follow existing test patterns and coverage thresholds
- Match existing code style (naming, formatting, patterns)
- Comments explain WHY, not WHAT
- PRs link to the plan/task they implement

## Communication
- Be concise and direct
- Report blockers immediately
- Proposals should be actionable, not theoretical
- Include file paths and line numbers in findings

## Backlog Preferences
- Keep proposals small and focused (1-3 day scope)
- Break large efforts into multiple proposals
- Include "why now" in every proposal — why is this worth doing?
- Security findings get their own proposal, never bundled with features
- Consolidate related small fixes into one proposal (don't flood the backlog)
- Max 5 open proposals, max 2 per day — quality over quantity
- If backlog is full, report findings in Slack and wait for reviews
