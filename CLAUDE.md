# RCao Claw - Claude Code Project Guide

## Architecture

RCao Claw is a secure, air-gapped AI development platform. It runs OpenClaw inside Docker with local LLM inference via Ollama (native on macOS for Metal GPU, or Docker on Linux) and delegates coding tasks to Claude Code through a locked-down SSH gateway.

**Key separation:**
- `rcao-claw/` = Claw's code (scripts, configs, Docker, personalities)
- `~/.openclaw/` = Runtime state per machine (managed by OpenClaw, not in repo)
- `$WORKSPACE/` = actual dev codebase (where Claude Code operates)

**Data flow:**
```
Browser -> localhost:3000 (Claw Web UI)
              |
              v
      Claw Gateway (Docker) --> Ollama LLM (native macOS / Docker Linux)
              |
              | SSH (ForceCommand, whitelisted only)
              v
      Host: ssh-gateway.sh -> allowed-commands.conf
              |
              +-- service-status.sh, git-status.sh, git-pull.sh
              +-- run-tests.sh, run-claude.sh
                      |
                      v
                  Claude Code (locked to $WORKSPACE)
```

## Ollama Modes

| Mode | When | How | GPU |
|------|------|-----|-----|
| `native` (default on macOS) | macOS with Apple Silicon | Ollama runs on host, container reaches via `host.docker.internal:11434` | Metal GPU (5-10x faster) |
| `docker` (default on Linux) | Linux / CI | Ollama runs in Docker container (profile `docker-ollama`) | CPU only |

Set via `OLLAMA_MODE=native|docker|auto` in `.env`. Auto-detected from OS.

## Security Layers (14 layers)

| # | Layer | What | Blocks |
|---|-------|------|--------|
| 1 | Docker | Process/filesystem isolation | Direct host access |
| 2 | 127.0.0.1 bind | Web UI on localhost only | Network exposure |
| 3 | SSH ForceCommand | Single entry point to host | Arbitrary command execution |
| 4 | ssh-gateway.sh | Command allowlist + logging | Unapproved scripts |
| 5 | Script arg validation | Input sanitization (no metacharacters, no `..`) | Injection attacks |
| 6 | Rate limiting | 30 commands/min per session | Brute force / abuse |
| 7 | Claude --permission-mode dontAsk | Tool whitelist mode | Unapproved tools |
| 8 | Claude --disallowedTools | Tool blacklist (ssh, curl, rm -rf, sudo, docker) | Dangerous tools |
| 9 | Claude --max-turns 25 | Execution limit | Runaway loops |
| 10 | Claude --max-budget-usd 10 | Cost cap | API cost overruns |
| 11 | .claude/settings.json | Persistent deny rules | Flag bypass |
| 12 | rbash user | Restricted shell for openclaw-bot | Shell escapes |
| 13 | Read-only mount | bin/ mounted :ro in Docker | Script tampering |
| 14 | Squid ACL proxy | HTTPS only to *.slack.com | Unauthorized internet |

## Network Architecture

Five Docker networks:
- **`isolated`** — `internal: true`. Ollama (docker mode) + Claw. No outbound internet.
- **`host-access`** — `internal: true`. Claw SSH to host via `host.docker.internal`.
- **`squid-internal`** — `internal: true`. Claw <-> Squid proxy.
- **`web-access`** — non-internal. Web UI port publishing to 127.0.0.1.
- **`squid-egress`** — non-internal. Squid outbound to Slack API only.

Port binding: `127.0.0.1:3000:3000` (Web UI, localhost only).

## Host Commands (via SSH Gateway)

| Command | Script | Purpose |
|---------|--------|---------|
| `git-status [repo]` | `bin/git-status.sh` | Check working tree |
| `git-pull [repo]` | `bin/git-pull.sh` | Pull latest with rebase |
| `run-tests [repo]` | `bin/run-tests.sh` | Run test suite |
| `run-claude <prompt> [repo]` | `bin/run-claude.sh` | Coding tasks (25 turns, $10 cap) |
| `service-status` | `bin/service-status.sh` | Host health + list repos |

Default workspace comes from `WORKSPACE` in `.env`. All commands accept an optional `[repo]` override (a name under the workspace parent, or a full path).

To add a new command: create `bin/<name>.sh`, add `<name>` to `bin/allowed-commands.conf`.

## Claude Code Lockdown

Managed via `config/claude-settings.json` and `bin/run-claude.sh` flags.

**Allowed tools:** Read, Edit, Write, Glob, Grep, npm (test/run/install/ci), git (read ops + add/commit/branch/checkout/stash), node (specific files), ls, mkdir, rm (single files by ext), wc, head, tail, sort, which.

**Blocked tools:** node -e/npx, curl/wget/ssh, sudo/docker, python/ruby/perl/awk/sed, rm -rf, bash -c/sh -c/eval/exec, git push/rebase/reset/merge/config, WebFetch, WebSearch.

**Limits:** 25 turns max, $10 budget per run.

## Docker Services

| Service | Container | Image | Resources | Networks |
|---------|-----------|-------|-----------|----------|
| `openclaw` | `rcao-claw` | Built from `docker/Dockerfile` | Dynamic (CLAW_MEM, CLAW_CPUS) | isolated, host-access, squid-internal, web-access |
| `ollama` (docker mode only) | `rcao-ollama` | `ollama/ollama:0.20.3` | Dynamic (OLLAMA_MEM, OLLAMA_CPUS) | isolated |
| `searxng` | `rcao-searxng` | `searxng/searxng:2026.3.28-cf5389afd` | 512M, 0.5 CPU | squid-internal |
| `valkey` | `rcao-valkey` | `valkey/valkey:8-alpine` | 128M, 0.25 CPU | squid-internal |
| `squid` | `rcao-squid` | `ubuntu/squid:latest` | 256M, 0.5 CPU | squid-internal, squid-egress |

Volume mounts:
- `~/.openclaw` → `/home/openclaw/.openclaw:rw` (runtime, managed by OpenClaw)
- `bin/` → `/openclaw/bin:ro` (scripts, immutable)
- `config/openclaw-docker-key` → `/openclaw/.ssh/id_ed25519:ro` (SSH key)
- `logs/` → `/openclaw/logs:rw` (audit trail)

## Shell Script Conventions

All scripts in `bin/`, `setup.sh`, `cleanup.sh`, `docker/entrypoint.sh` follow:
- `set -euo pipefail` at top
- Source `workspace-env.sh` for workspace resolution (in bin/ scripts)
- Parse `.env` safely with `grep`/`cut` — never `source .env`
- Validate inputs: reject shell metacharacters (``; | & $ ` \ ( ) { } < >``), path traversal (`..`), absolute paths
- Log actions to `~/openclaw/logs/` with ISO timestamps
- Use colored output helpers: `info()`, `warn()`, `error()`, `step()`

## Commit Conventions

- Conventional commits: `feat:`, `fix:`, `docs:`, `chore:`
- One concern per PR
- Feature branch from `main` — never push to main directly
- ShellCheck must pass on all `.sh` files
- Test full setup/cleanup cycle if changing `setup.sh` or `cleanup.sh`
